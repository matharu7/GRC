
# 🛡️ GRC Domain (Governance, Risk & Compliance)

---

## 🔹 1. Introduction to GRC

👉 **GRC = Governance + Risk + Compliance**

| Component  | Meaning                                          |
| ---------- | ------------------------------------------------ |
| Governance | Define policies, rules, and processes            |
| Risk       | Identify, analyze, and manage risks              |
| Compliance | Follow legal, regulatory, and industry standards |

---

## 🔥 2. Practical Work in GRC (Real Industry Tasks)

---

## 🔸 2.1 Policy Creation (Daily Activity)

```text
Tasks:
- Create security policies
- Define organizational rules
- Maintain documentation
```

### Examples:

* Password Policy
* Access Control Policy
* Data Protection Policy

👉 **Real Job Insight:**
GRC professionals spend significant time writing and maintaining **formal security documents**.

---

## 🔸 2.2 Risk Assessment (Core Responsibility)

```text
Tasks:
- Identify risks
- Analyze impact and likelihood
- Recommend security controls
```

### Examples:

* Public-facing server → Security Risk
* Weak password policy → Risk

### Framework Used:

* National Institute of Standards and Technology (NIST Risk Framework)

---

## 🔸 2.3 Compliance Implementation

```text
Tasks:
- Map internal controls to standards
- Ensure organization meets compliance requirements
- Perform validation and documentation
```

### Common Standards:

* International Organization for Standardization ISO 27001
* General Data Protection Regulation GDPR
* Payment Card Industry Data Security Standard PCI DSS

👉 **Work Nature:** Checklist-based validation + documentation

---

## 🔸 2.4 Security Audits

```text
Tasks:
- Evaluate security controls
- Identify compliance gaps
- Recommend improvements
```

### Types:

* Internal Audit
* External Audit

### Examples:

* Log verification
* Access control review
* Encryption validation

---

## 🔸 2.5 Risk Register Management

```text
Tasks:
- Maintain centralized risk list
- Track mitigation progress
```

### Example:

| Risk          | Severity | Status |
| ------------- | -------- | ------ |
| Weak Password | High     | Fixed  |

---

## 🔸 2.6 Vendor / Third-Party Risk Management

```text
Tasks:
- Assess vendor security posture
- Evaluate third-party risks
```

### Examples:

* Cloud service provider security
* Payment gateway compliance

---

## 🔸 2.7 Incident & Compliance Reporting

```text
Tasks:
- Report security incidents
- Maintain compliance documentation
```

### Example:

* Data breach reporting (within 72 hours under GDPR)
